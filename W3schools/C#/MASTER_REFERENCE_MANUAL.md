## 1. Scope & Learning Objectives

The C# curriculum provides a comprehensive introduction to the **.NET ecosystem** and the C# language — a statically-typed, managed-code language widely utilized in enterprise applications, game development (Unity), and web services. The course serves as an essential bridge for understanding managed runtime environments and object-oriented software architecture.

This manual documents the modules completed, the concepts mastered, and the objective validation of competency achieved through the curriculum's final assessment.

---

## 2. Curriculum Architecture

### 2.1 Core Language Syntax

Foundational elements of C# development are covered in the initial modules:

- **Variables & Data Types:** Strongly-typed variable declaration, value types vs. reference types, and built-in primitives.
- **Operators:** Arithmetic, relational, logical, and assignment operators within the C# type system.

These fundamentals establish the syntactic baseline for all subsequent object-oriented and advanced language features.

### 2.2 Object-Oriented Programming

The curriculum places significant emphasis on OOP as the core architectural paradigm of C#:

- **Classes & Objects:** Encapsulation of data and behavior within class definitions; instantiation and lifecycle management.
- **Inheritance:** Deriving new classes from base classes to promote code reuse and establish hierarchical type relationships.
- **Polymorphism:** Method overriding, virtual methods, and runtime dispatch enabling flexible and extensible designs.
- **Interfaces:** Contract-based abstraction enabling multiple behavioral contracts and decoupled component design.

These concepts provide a strong basis for modular, maintainable software design in the .NET ecosystem.

### 2.3 Advanced Constructs

The curriculum introduces language features critical for modern data manipulation:

- **Lambda Expressions:** Anonymous functions enabling concise inline logic and functional-style programming.
- **LINQ (Language Integrated Query):** A unified query syntax for querying collections, databases, and XML — seamlessly integrated into the C# language.
- **Collections:** Generic and non-generic collection types (`List<T>`, `Dictionary<K,V>`, etc.) for efficient data organization and manipulation.

These constructs are essential for writing expressive, data-oriented C# code in production environments.

### 2.4 Exception Handling

Strategies for robust software development are covered through:

- **Try-Catch-Finally Blocks:** Structured error handling for capturing and responding to runtime exceptions.
- **Exception Types:** Understanding the hierarchy of system and application-defined exceptions.
- **Throwing Exceptions:** Explicitly signaling error conditions to calling code.

Proper exception handling is a hallmark of professional-grade application development.
