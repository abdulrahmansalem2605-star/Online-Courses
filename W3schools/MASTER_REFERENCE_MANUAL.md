## 3. C Programming Language

### 3.1 Scope & Learning Objectives

The C curriculum provides a structured introduction to the procedural programming paradigm. It is designed to transition a learner from basic syntactic familiarity to a working understanding of systems-level programming concepts. The course establishes the foundational vocabulary for communicating with the machine at a level close to the hardware.

### 3.2 Curriculum Architecture

**3.2.1 Core Syntax & Control Structures**  
Data types (`int`, `char`, `float`, `double`), operators (arithmetic, relational, logical, bitwise, assignment), and conditional logic (`if-else`, `switch`).

**3.2.2 Looping Mechanisms**  
Three iteration constructs: `for` (determinate entry-controlled), `while` (indeterminate entry-controlled), and `do-while` (exit-controlled, guaranteed minimum one execution).

**3.2.3 Functional Programming**  
Function declaration, definition, parameter passing (pass-by-value semantics), and return values as the primary unit of code organization.

**3.2.4 Memory & Data Management**  
Arrays (contiguous allocation), strings (null-terminated character arrays), and **pointers** — direct memory addressing, pointer arithmetic, and pointer-array relationships. Pointers are the critical differentiator between C and higher-level languages, enabling direct memory manipulation.

**3.2.5 Custom Data Structures & File I/O**  
`struct` definitions for composite data types and standard I/O functions for persistent file storage.

## 4. C++ Programming Language

### 4.1 Scope & Learning Objectives

The C++ curriculum serves as a fundamental bridge between procedural programming (as established in C, its predecessor) and Object-Oriented Programming. It provides a structured pathway for understanding the enhanced capabilities C++ offers over C — retaining systems-level control while introducing abstraction mechanisms for large-scale software design.

### 4.2 Curriculum Architecture

**4.2.1 Object-Oriented Programming Fundamentals**  
Classes and objects (encapsulation of data and behavior), inheritance (hierarchical type relationships), polymorphism (virtual functions, overloading), and encapsulation (access specifiers `public`, `protected`, `private`). These are essential for software architecture in AI projects.

**4.2.2 Standard Template Library (STL)**  
Vectors (dynamic arrays), iterators (generalized pointer abstractions for traversal), and containers (`map`, `set`, `list`, etc.) for diverse data management. The STL is vital for efficient data handling in modern C++.

**4.2.3 Memory Management & Advanced Syntax**  
References (aliases providing safer indirection than raw pointers), pointers (direct memory addressing), and dynamic memory allocation via `new` and `delete`.

**4.2.4 Modern C++ Features**  
Exposure to enhanced standard libraries and syntax improvements that streamline complex coding tasks and improve code safety.

## 5. C# Programming Language

### 5.1 Scope & Learning Objectives

The C# curriculum provides a comprehensive introduction to the .NET ecosystem and the C# language — a statically-typed, managed-code language widely utilized in enterprise applications, game development (Unity), and web services. It serves as an essential bridge for understanding managed code and object-oriented architectures.

### 5.2 Curriculum Architecture

**5.2.1 Core Language Syntax**  
Variables and data types (strongly-typed declaration, value vs. reference types), and operators within the C# type system.

**5.2.2 Object-Oriented Programming**  
Classes and objects (encapsulation, lifecycle), inheritance (code reuse, type hierarchies), polymorphism (method overriding, virtual methods, runtime dispatch), and interfaces (contract-based abstraction, decoupled design).

**5.2.3 Advanced Constructs**  
Lambda expressions (anonymous functions for functional-style programming), LINQ (Language Integrated Query for unified data querying), and collections (`List<T>`, `Dictionary<K,V>`, etc.).

**5.2.4 Exception Handling**  
`try-catch-finally` blocks, exception type hierarchies, and explicit exception signaling — strategies for robust software development.

## 6. Java Programming Language

### 6.1 Scope & Learning Objectives

The Java curriculum provides a systematic introduction to one of the most widely used enterprise-level languages. It explores the principles of the Java Virtual Machine (JVM) environment, strong typing, and the extensive Java ecosystem — a cornerstone of large-scale software development. Java's strict adherence to OOP makes this study critical for understanding architectural design.

### 6.2 Curriculum Architecture

**6.2.1 Core Java Syntax**  
Primitive data types (`int`, `double`, `boolean`, `char`), control flow (`if-else`, `switch`, loops), and operators within Java's type-safe environment.

**6.2.2 Object-Oriented Programming Excellence**  
Inheritance (single-inheritance hierarchies via `extends`), polymorphism (runtime overriding, compile-time overloading), abstraction (abstract classes, interfaces), encapsulation (access modifiers `private`, `protected`, `public`).

**6.2.3 Java Collections Framework**  
Core interfaces (`List`, `Set`, `Queue`, `Map`), concrete implementations (`ArrayList`, `HashSet`, `HashMap`, `LinkedList`) with distinct performance characteristics, directly applicable to data structures and algorithmic efficiency studies.

**6.2.4 Exception Handling & Multithreading**  
`try-catch-finally` for resilient applications (checked vs. unchecked exceptions); basic thread creation via `Thread` class and `Runnable` interface for concurrency.

## 7. Cross-Domain Synthesis

### 7.1 Paradigm Progression

The four languages form a natural progression from low-level to high-level abstraction:

1. **C** — Procedural, manual memory management, hardware-near
2. **C++** — Multi-paradigm, retains C memory control while adding OOP abstraction
3. **C#** — Fully managed runtime, garbage-collected, component-oriented
4. **Java** — Fully managed JVM runtime, garbage-collected, strictly OOP

This progression mirrors the evolution of systems and application programming, providing a comprehensive understanding of how different levels of abstraction and memory management strategies affect software design.

### 7.2 Common OOP Foundation Across Managed Languages

C++, C#, and Java all share the four pillars of OOP (inheritance, polymorphism, encapsulation, abstraction), though each enforces them differently:
- **C++** provides opt-in OOP with manual control over vtables and memory.
- **C#** provides comprehensive OOP with a unified type system and properties.
- **Java** provides strictly-enforced OOP with single inheritance and interface contracts.

This shared foundation enables rapid cross-language transfer while the differences sharpen architectural judgment.
