## 1. Scope & Learning Objectives

The C++ curriculum provides a structured pathway from **procedural programming** to **Object-Oriented Programming (OOP)** . As the direct successor to C, C++ retains systems-level control while introducing abstraction mechanisms that enable large-scale software design. This course establishes the foundation for understanding how high-performance, object-oriented systems are architected — a critical competency for AI and systems engineering.

This manual documents the modules completed, the concepts mastered, and the objective validation of competency achieved through the curriculum's final assessment.

## 2. Curriculum Architecture

### 2.1 Object-Oriented Programming Fundamentals

The curriculum's core focus is on the four pillars of OOP, which are essential for software architecture in AI projects:

- **Classes & Objects:** User-defined types that encapsulate data members and member functions; instantiation and object lifecycle.
- **Inheritance:** Hierarchical relationships between classes enabling code reuse and polymorphic behavior.
- **Polymorphism:** Virtual functions, function overloading, and operator overloading — enabling a single interface to operate on different underlying types.
- **Encapsulation:** Access specifiers (`public`, `protected`, `private`) controlling visibility and protecting internal state.

These OOP principles provide the architectural vocabulary for designing modular, extensible software systems.

### 2.2 Standard Template Library

The STL is introduced as a powerful library of generic data structures and algorithms:

- **Vectors:** Dynamic arrays providing resizable storage with contiguous memory layout.
- **Iterators:** Generalized pointer abstractions enabling traversal of container elements.
- **Containers:** Additional associative and sequential containers (`map`, `set`, `list`, etc.) for diverse data management requirements.

The STL is vital for efficient data handling and is a cornerstone of modern C++ development.

### 2.3 Memory Management & Advanced Syntax

Building on C's memory model, C++ introduces refined mechanisms:

- **References:** Aliases to existing variables providing safer indirection than raw pointers.
- **Pointers:** Continued use of direct memory addressing with additional C++-specific features.
- **Dynamic Memory:** Operators `new` and `delete` for heap allocation and deallocation.

These features allow for high-performance software development with explicit control over resource management.

### 2.4 Modern C++ Features

The curriculum exposes features introduced in modern C++ standards that streamline complex coding tasks:

- **Standard Libraries:** Enhanced standard library components that reduce boilerplate and improve code safety.
- **Syntax Improvements:** Language features that enable more expressive and concise implementations.
